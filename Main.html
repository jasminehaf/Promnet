Git Topic

Git adalah sistem kontrol versi (Version Control System) yang dibuat oleh Linus Torvalds yang bertugas untuk mencatat perubahan seluruh file atau repository suatu project. Git diciptakan dengan tujuan untuk mengelola perubahan dalam kode sumber suatu proyek dengan cara yang efisien dan efektif. Git juga dikenal sebagai sistem pengontrol revisi terdistribusi (Distributed Version Control System atau DVCS), yang berarti database Git tidak terpusat di satu tempat saja. Semua orang yang terlibat dalam pengembangan proyek akan memiliki salinan database Git, yang mempermudah pengelolaan proyek baik saat terhubung ke internet maupun dalam keadaan offline. Selain itu, Git secara aktif memonitor semua perubahan yang terjadi pada file proyek dan menyimpannya dalam database.

Fungsi Git

1. Berkolaborasi. Git digunakan untuk bekerja sama dengan banyak orang dalam proyek yang sama, memungkinkan tim untuk bekerja secara efisien dan mengelola perubahan kode dengan mudah.

2. Proyek Open Source. Git sangat cocok untuk proyek open source, di mana banyak kontributor dapat bekerja secara independen dan menggabungkan kode mereka dengan mudah.

3. Membantu Organisir. Git membantu mengatur dan mengelola versi source code program dengan memberikan tanda baris dan kode mana yang perlu ditambahkan atau diganti.

4. Platform Fleksibilitas. Git dapat digunakan dalam berbagai bahasa pemrograman dan lingkungan pengembangan, menjadikannya alat yang fleksibel untuk berbagai jenis proyek

5. Menjadi Backup. Git juga berfungsi sebagai backup untuk proyek pengembangan perangkat lunak, memastikan bahwa kode yang berharga tidak hilang atau rusak

Git Command

  • Git init: untuk membuat repository baru atau reinisialisasi repository yang sudah ada
  • Git status: untuk mengecek status dari repository lokal
  • Git log: untuk melihat riwayat commit
  • Git add: untuk menambahkan file ke index
  • Git commit: untuk commit pada perubahan ke head
  • Git push: untuk mengirim perubahan yang telah di-commit ke remote repository
  • Git config: untuk mengatur konfigurasi git seperti nama pengguna dan alamat email
  • Git clone: untuk mengunduh sebuah repository dan seluruh riwayat revisinya
  • Git branch: untuk membuat, menghapus, atau menampilkan daftar branch dalam repository
  • Git checkout: untuk beralih antara branch yang ada dalam repository
  • Git merge: untuk menggabungkan perubahan dari satu branch ke branch lainnya
  • Git diff: untuk menampilkan perbedaan antara file yang ada dalam repository dengan versi sebelumnya
  • Git fetch: untuk mengambil commit terbaru dari remote repository dan menyimpannya di branch origin/master tanpa langsung melakukan merge dengan branch aktif
  • Git pull: untuk mengambil commit terbaru dari remote repository dan menggabungkan commit tersebut dengan branch aktif
